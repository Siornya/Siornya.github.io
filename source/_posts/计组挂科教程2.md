## 总线的基本概念

总线是连接多个部件的信息传输线，是各个部件共享的传输介质。

在某一时刻只能有一个部件向总线发送信息，但可以有多个部件同时从总线上接收相同的信息。

*对于集成工业糕手，可以想象有一条传送效率无限高的传送带，能够连接一个仓库作为输入，而传送带上的每个设备都可以从中获取材料*

*此外也很容易想象总线的优点，由于我们显然不能造大量的设备，所以每个设备作为某一产线中不的某一步，需要和所有其他设备连接，理论上需要 $\frac{n(n-1)}{2}$ 条连线，而总线可以将其优化成一组*

### 总线的两个本质特性：共享 + 分时

这是理解总线的关键，务必记牢：

- **共享（Share）**：多个部件连接在同一组线上，这组线是大家公用的。
- **分时（Time-sharing）**：正因为是公用的，所以**同一时刻只能有一个部件往总线上"说话"（发送数据）**，其他部件只能"听"或者等。

正因为有"分时"这个限制，后面才会引出**总线仲裁**和**通信定时**。

## 总线的分类

1. 片内总线：芯片内部的总线
2. 系统总线：计算机各部件的信息传输线
   - 数据总线：双向，与机器字长、存储字长有关
   - 地址总线：单向，与存储地址，I/O地址有关
   - 控制总线：有出，有入
3. 通信总线：用于计算机系统之间或计算机系统与其他系统之间的通信
   - 串行通信总线
   - 并行通信总线

| 层次 | 连接谁和谁 | 举例 |
|------|-----------|------|
| **片内总线** | 芯片内部各部件（如 CPU 内部寄存器与 ALU 之间） | CPU 芯片内部 |
| **系统总线** | CPU、主存、I/O 接口三大件之间 | 主板上的总线 |
| **通信总线** | 计算机与计算机、计算机与外部系统之间 | 网络、设备间通信 |

**按数据传送方式分类**

- **串行总线**：数据一位一位地依次传（一根数据线）。线少、抗干扰好、适合长距离，但慢。例：USB。
- **并行总线**：多位数据同时传（多根数据线并排）。一次传得多、快，但线多、长距离时各线之间会互相干扰。

### 系统总线的三大组成

系统总线按传送的信息种类，分成三组线。记成 **"数据、地址、控制"**

### 数据总线 DB（Data Bus）
- 作用：传送**真正要处理的数据**（也包括指令）。
- 方向：**双向**（数据可以从 CPU 到内存，也可以从内存回 CPU）。
- 位数（宽度）：通常等于**机器字长**或**存储字长**。
- 意义：数据总线有多少根，决定了**一次能并行传多少位数据**，直接影响速度。

### 地址总线 AB（Address Bus）
- 作用：传送**地址**，即告诉对方"我要访问内存/设备的哪个位置"。
- 方向：**单向**（地址只由 CPU 发出，指向内存或外设）。
- 位数：决定了**最大寻址范围**。地址线有 n 根，就能表示 2ⁿ 个地址。
  - 例：20 根地址线 → 可寻址 2²⁰ = 1M 个单元。
- 记忆：**地址总线宽度决定"能管多大的内存"。**

### 控制总线 CB（Control Bus）
- 作用：传送**控制信号、时序信号、状态信号**，相当于"指挥棒"。
- 方向：**对每一根来说是单向，但整体有出有入**——有的信号是 CPU 发出去的（如读、写命令），有的是设备反馈回来的（如中断请求、就绪信号）。
- 常见信号：存储器读/写、I/O 读/写、中断请求/响应、总线请求/允许、时钟、复位等。

> 类比一次寄快递：**地址总线**=收件地址（送到哪），**数据总线**=包裹本身（送什么），**控制总线**=快递流程指令（"取件""已签收""派送中"这些状态和命令）

## 总线特性与性能指标

### 总线特性

1. **机械特性**：尺寸，形状，引脚数量和排列顺序
2. **电气特性**：传输方向和有效的电平范围
3. **功能特性**：每根传输线的功能（地址/数据/控制）
4. **时间特性**：各信号之间的时序关系

### 总线的性能指标

- **总线宽度**：数据总线的根数（多少位）
- **总线带宽**：每秒传输的最大字节数
- **时钟同步/异步**
- **总线复用**：让同一组线在不同时间分别当**地址线**和**数据线**用
- **信号线数**：地址线、数据线和控制线根数的总和
- **总线控制方式**：并发、自动、仲裁、逻辑、计数
- **总线（工作）频率**：总线一秒能工作多少次，单位 MHz

带宽计算公式 `总线带宽 = 总线宽度(位) ÷ 8 × 总线工作频率`

### 总线标准

- **ISA / EISA**：早期 PC 扩展总线，现已淘汰。
- **PCI**：经典的并行系统总线。
- **PCI-E（PCIe）**：现代主流，串行、高速、点对点，显卡/固态硬盘都在用。
- **AGP**：曾专供显卡，已被 PCIe 取代。
- **USB**：通用串行总线，热插拔、即插即用，外设标配。

## 总线结构

随着设备速度差异变大，总线结构一步步进化，核心矛盾是：**慢设备会拖累快设备**。

**单总线结构**
CPU、主存、所有 I/O 设备**全挂在同一条系统总线上**。

- 优点：结构简单、成本低、易扩展。
- 缺点：所有传输都抢这一条线，**带宽是瓶颈**；快设备（CPU、内存）被慢设备（外设）拖慢；不能并发传输。

**双总线结构**

把负担分开，常见两种思路：
- **面向 CPU 的双总线**：CPU 与主存一条总线，CPU 与 I/O 一条总线。缺点是 I/O 和主存交换数据都得经过 CPU，CPU 很累。
- **面向存储器的双总线**：额外增加一条"存储总线"专供 CPU 和主存高速交换，其余设备走系统总线。减轻了主存访问压力。

**三总线结构**

分成三条专用总线：
- **主存总线**：CPU 与主存之间。
- **I/O 总线**：CPU 与各类 I/O 设备之间。
- **DMA 总线**：高速外设（如磁盘）与主存之间**直接**传数据，不经过 CPU（DMA = 直接存储器访问）。

好处：高速外设和内存直传时，CPU 可以同时干别的事，提高了并行度。

**多总线 / 桥接结构（现代主流思想）**

用**总线桥（Bridge）**把不同速度的总线连起来，各跑各的速度：
- **CPU 总线（局部总线）**：最快，连 CPU 和 Cache 等。
- **系统总线 / PCI 总线**：中速，连主存、高速扩展设备。
- **扩展总线 / 低速总线**：连慢速外设。

> 核心思想：**让快的归快、慢的归慢，用桥隔开，互不拖累**。这就是现代计算机总线设计的主流。

## 总线控制

### 总线判优控制

"分时"决定了同一时刻只能一个部件占用总线。当多个部件**同时**想用总线时，必须有个机制决定**让谁先用**——这就是仲裁。

- 主设备：拥有总线控制权，决定何时传输、传输什么
- 从设备：响应从主设备发来的总线命令

*主设备决定接下来要执行的内容，不一定是写数据的设备*

有两大判优方式：集中式和分布式

集中式包括链式查询，计数器定时查询和独立请求三种方式

**链式查询**

用 3 根关键线：**BR（总线请求）**、**BG（总线同意）**、**BS（总线忙）**。

- BG（同意信号）从仲裁器出来后，**像串糖葫芦一样串行经过每个设备**：第一个设备如果想用就截下 BG 自己用，不用就传给下一个。
- **离仲裁器越近的设备，优先级越高**。
- ✅ 优点：用线最少，结构最简单，最容易增加设备。
- ❌ 缺点：优先级固定；对**线路故障敏感**（链条中间断一个，后面全瘫）；查询速度慢。

**计数器定时查询**

- 仲裁器里有个**计数器**，通过一组**设备地址线**轮流"点名"各设备，点到谁的编号且谁正好在请求，就把总线给谁。
- 优先级**可灵活设置**：
  - 计数值每次从 **0** 开始 → 固定优先级（编号小的优先）。
  - 计数值从**上次停的地方**接着开始 → 循环优先级（更公平）。
- ✅ 优点：优先级灵活，对故障不那么敏感。
- ❌ 缺点：增加了计数和地址线，控制比链式复杂。

**独立请求方式**

- **每个设备各有一对独立的 BR 和 BG 线**，直接连到仲裁器。
- 仲裁器内部有**排队电路**，统一决定把总线给谁。
- ✅ 优点：响应**最快**；优先级安排**最灵活**（想怎么排都行）。
- ❌ 缺点：**用线最多**，控制电路最复杂、成本最高。

| 方式 | 用线数量 | 速度 | 优先级灵活性 | 对故障敏感度 |
|------|---------|------|------------|------------|
| 链式查询 | 最少 | 最慢 | 固定，不灵活 | 最敏感 |
| 计数器查询 | 中等 | 中等 | 较灵活 | 中等 |
| 独立请求 | 最多 | 最快 | 最灵活 | 最不敏感 |

从链式到独立请求，是**"线越用越多 → 换速度越快、越灵活"**的过程

**分布式仲裁（了解）**

没有中央仲裁器，每个设备自带仲裁号和仲裁电路，通过相互通信自行决定优先级，常见于多处理器系统

### 总线通信控制

通信控制解决双方协调配合问题，"用的时候双方怎么对上节奏"——因为 CPU 快、外设慢，必须协调好"我发"和"你收"的时机。

#### 总线传输周期
1. **申请分配阶段**：主设备请求，经仲裁获得总线控制权
2. **寻址阶段**：主设备发出要访问的地址和命令
3. **传数阶段**：主从设备之间传输数据
4. **结束阶段**：主模块撤除有关消息，让出总线

#### 总线通信的四种方式

**同步通信（同步定时）**

- 用**统一的时钟信号**，所有动作都卡在固定的时钟节拍上完成。
- ✅ 优点：规整、简单、控制方便、传输**速度快**。
- ❌ 缺点：**强制同步**，时钟节拍必须照顾**最慢的设备**来定，导致快设备也得等，整体效率被拉低；不够灵活。
- 适用：总线长度短、各设备**速度比较接近**的场合。

**异步通信（异步定时）**

- **没有统一时钟**，靠主从双方互相发"**握手 / 应答信号**"来协调：主方发**请求（Request）**，从方处理好后回**应答（Acknowledge）**。
- 谁快谁慢都能配合，按各自实际速度走。
- 根据请求与应答信号的"互锁"程度，分三种：
  - **不互锁**：主方发请求后，不等应答、过一段时间就自动撤请求；从方应答也类似。最快但最不可靠。
  - **半互锁**：主方发请求后，**必须等到应答**才撤请求；但从方应答不等主方撤请求就自己撤。可靠性居中。
  - **全互锁**：请求要等应答才撤，应答也要等请求撤了才撤，**完全互相确认**。最可靠，但最慢。
- ✅ 优点：灵活，能适应速度差异很大的设备。
- ❌ 缺点：需要握手，控制复杂，速度通常比同步慢。

**半同步通信**

- **同步 + 异步的结合**：保留统一时钟（同步的骨架），但增加一根 **WAIT（等待）信号**。
- 当从设备速度跟不上时，就拉住 WAIT 信号，让主设备**插入若干等待时钟周期**，等从设备准备好再继续。
- 既保留了同步的规整，又能照顾慢设备，是一种折中。

**分离式通信（最大化利用总线）**

- 把一次传输拆成两个**子周期**，中间**释放总线**：
  - **子周期 1**：主模块申请总线，发出地址和命令后**立刻释放总线**（不傻等从模块准备数据）。
  - **子周期 2**：从模块数据准备好后，**重新申请总线**，把数据发回去。
- ✅ 优点：在从设备"准备数据"的空档里，总线可以被别的设备使用，**总线利用率最高**。
- ❌ 缺点：控制最复杂，每个模块既要能当主又要能当从。

**四种定时方式对比：**

| 方式 | 是否用统一时钟 | 特点 | 速度/效率 | 复杂度 |
|------|--------------|------|----------|--------|
| 同步 | 是 | 卡固定节拍 | 快但迁就慢设备 | 简单 |
| 异步 | 否 | 握手应答 | 灵活但较慢 | 较高 |
| 半同步 | 是（+等待信号） | 同步为主可插等待 | 折中 | 中 |
| 分离式 | — | 拆两段、传输间释放总线 | 利用率最高 | 最高 |

## 总结

```
系统总线
├── 是什么：公共信息传输线 → 特性：共享 + 分时
├── 为什么：取代点对点连线，省线、易扩展
├── 分类
│   ├── 按方式：串行 / 并行
│   └── 按层次：片内 / 系统 / 通信
├── 三大组成（系统总线）
│   ├── 数据总线 DB —— 双向，宽度≈字长，定"传多少"
│   ├── 地址总线 AB —— 单向，n根→2ⁿ寻址，定"管多大内存"
│   └── 控制总线 CB —— 有出有入，发命令传状态
├── 四特性：机械 / 电气 / 功能 / 时间
├── 性能指标：宽度、频率、带宽
│   └── 带宽 = (宽度/8) × 频率
├── 结构演变：单总线 → 双总线 → 三总线 → 多总线(桥)
│   └── 核心：快慢分离，互不拖累
├── 仲裁（谁来用）：主设备 vs 从设备
│   ├── 集中式：链式查询 / 计数器查询 / 独立请求
│   │            （省线慢 ←——————→ 费线快灵活）
│   └── 分布式：各自竞争，比较仲裁号
└── 通信定时（怎么对节奏）：四阶段
    ├── 同步：统一时钟，快但迁就慢设备
    ├── 异步：握手应答，分 不/半/全 互锁
    ├── 半同步：同步 + WAIT 等待信号
    └── 分离式：拆两段、传输间释放总线，利用率最高
```
