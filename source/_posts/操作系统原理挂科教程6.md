## I/O系统

从OS的角度看待硬件，其性能指标有:设备使用特性、数据传输速率、数据的传输单位，设备共享属性等。

```
CPU / 内存
   │
I/O 通道（大型机）/ 总线
   │
设备控制器（I/O 控制器、设备适配器）
   │
设备本身（机械部分）
```

### 设备的分类

| 分类标准 | 类别 | 说明 |
|---|---|---|
| 按**信息传输单位** | **块设备** | 以"数据块"为单位传输，可寻址、可随机访问，如磁盘 |
| | **字符设备** | 以"字符"为单位传输，不可寻址、只能顺序访问，如键盘、打印机 |
| 按**使用特性** | 存储设备 | 用于存储信息，如磁盘、磁带 |
| | I/O 设备 | 输入设备（键盘、鼠标）、输出设备（显示器、打印机） |
| 按**传输速率** | 低速 / 中速 / 高速 | 键盘（低速）→ 打印机（中速）→ 磁盘（高速） |
| 按**共享属性** | **独占设备** | 一段时间只能给一个进程用，如打印机 |
| | **共享设备** | 可同时（交替）供多个进程使用，如磁盘 |
| | **虚拟设备** | 用 SPOOLing 技术把独占设备改造成可共享的逻辑设备 |

### 设备控制器

设备控制器是计算机中的一个实体，控制一个或多个I/O设备，实现设备和计算机之间的数据交换。也称为适配器，是CPU和I/O设备之间的接口。通常是一块印刷电路板。

主要功能：

- **接收和识别 CPU 命令**：CPU 把命令写入控制器的**命令寄存器**，控制器解析后驱动设备执行。
- **数据交换**：在 CPU（内存）与设备之间传送数据，靠**数据寄存器**暂存。
- **标识和报告设备状态**：用**状态寄存器**记录设备的忙/闲、就绪、出错等状态，供 CPU 查询。
- **地址识别**：每个设备/寄存器都有地址，控制器要能识别 CPU 当前操作的是哪个设备、哪个寄存器。
- **数据缓冲**：解决I/O设备与CPU速度不匹配的问题。
- **差错检测**：对传输的数据做校验，发现传输或设备本身的错误。

设备控制器实现了处理器与设备之间的数据、控制和地址信息等信息的传输。

设备控制器由控制/状态寄存器、数据寄存器、与处理机接口、与设备接口等构成。

CPU 与控制器寄存器通信有两种编址方式：**内存映射 I/O**（寄存器与内存统一编址，用访存指令即可读写）和**独立编址**（I/O 端口用专门的 I/O 指令访问）。

### I/O通道

I/O 通道是一种**特殊的处理机**，主要出现在大型机中。它的引入是为了进一步减少 CPU 对 I/O 的干预：把原本由 CPU 完成的 I/O 控制工作交给通道，让 CPU 与 I/O 并行。

- 通道有自己的指令系统（通道指令较简单、专用于 I/O），能执行由这些指令构成的**通道程序**。
- CPU 只需发一条"启动通道"指令，指明通道程序的内存地址，通道便独立完成一整批读写，全部完成后才向 CPU 发一次中断。
- **通道 ≠ 一般处理机**：它没有自己的内存，**通道程序放在主存中**、与 CPU 共享内存；指令类型单一，只能做 I/O 控制。

通道与 DMA 的区别：DMA 一次只能传一个连续数据块、且每次传输的内存地址和大小由 CPU 设置；**通道可一次处理一组数据块**（执行一段通道程序），CPU 干预更少、并行性更高。

通道按信息交换方式，通道分三类：

| 类型 | 特点 | 适用设备 |
|---|---|---|
| **字节多路通道** | 以字节为单位，分时共享的方式传输数据 | 键盘、打印机等慢速字符设备 |
| **数组选择通道** | 一段时间内只为一台高速设备独占服务，传完才换下一台 | 磁盘等高速设备 |
| **数组多路通道** | 综合前两者：以数据块为单位，多台高速设备分时共享，效率高 | 多台磁盘并行 |

## I/O控制方式

I/O 控制方式的发展在不断减少CPU对 I/O 的干预，提高并行性

**可编程I/O 程序直接控制**

- CPU 与 I/O 完全串行，CPU 绝大部分时间在空转"踏步等待"（忙等），效率极低。
- 一段时间内，CPU只能和一台设备交换数据
- 无中断机制

---

**中断驱动方式**

CPU 发出 I/O 命令后不再等待，转去执行别的进程；设备完成一个字的传输后，通过中断通知 CPU；CPU 响应中断、取走数据，再启动下一次传输。

- CPU 与 I/O 可以并行，效率比轮询高很多。

- **缺点**：仍是以"字（字节）"为单位中断，每传一个字就中断一次，数据量大时中断太频繁，CPU 频繁切换、开销大。

---

**DMA方式**

引入DMA控制器，让数据以"数据块"为单位直接在设备和内存之间传输，整块传输期间不需要 CPU 干预。

- CPU 只在传输开始时给出命令（要传哪个块、内存地址、字数），在传输结束时收到一次中断。中间的逐字搬运由 DMA 控制器直接完成，不经过 CPU。
- DMA 控制器中的寄存器：**命令/状态寄存器 CR**、**内存地址寄存器 MAR**（数据在内存的地址）、**数据寄存器 DR**、**数据计数器 DC**（还要传多少字）。
- **特点**：以块为传输单位；数据直接在内存与设备间传送（不经 CPU 寄存器中转）；仅在一个数据块传完时才中断一次。
- **缺点**：一次只能传输一个连续的数据块；多个离散数据块仍需 CPU 多次干预。

**通道控制方式**

I/O 通道是 DMA 的进一步发展，能一次处理一组数据块。CPU 只需发一条"启动通道"指令，指明要执行的通道程序，通道便独立完成一系列读写，全部完成后才向 CPU 发一次中断。

- 传输单位：一组数据块。
- CPU 干预最少：一次 I/O 操作（含多个块）只中断一次。
- 实现了 CPU、通道、设备三者的高度并行。

---

**四种方式对比**

| 控制方式 | 传输单位 | CPU 干预程度 | 中断频率 | 数据流向 |
|---|---|---|---|---|
| 程序直接控制 | 字 | 全程忙等 | 不用中断（轮询） | 经过 CPU |
| 中断驱动 | 字 | 每字中断 | 每个字一次 | 经过 CPU |
| DMA | 块 | 块首/块尾 | 每个块一次 | 设备↔内存直达 |
| 通道 | 一组块 | 极少 | 一组块一次 | 设备↔内存直达 |

## 缓冲管理

在设备和 CPU/内存之间设置**缓冲区**，主要为了：

1. **缓和 CPU 与 I/O 设备速度不匹配的矛盾**
2. **减少对 CPU 的中断频率**，放宽对中断响应时间的限制（攒一批再中断）
3. **解决数据大小不匹配**的问题
4. **提高 CPU 与 I/O 设备之间的并行性**

缓冲区可以用硬件实现，但更常见的是在**内存中划出一块区域**作软件缓冲。

### 缓冲技术的几种实现

- **单缓冲**：设一个缓冲区。设备把数据送入缓冲区，CPU 再从缓冲区取走。同一时刻缓冲区要么在装入、要么在取出，CPU 与设备对同一缓冲区的操作不能完全并行。
- **双缓冲（缓冲对换）**：设两个缓冲区。设备往缓冲区 1 装数据的同时，CPU 从缓冲区 2 取数据；装满/取空后两者交换角色。并行度比单缓冲高，可实现双向同时传输。
- **循环缓冲**：把多个缓冲区组织成环形队列，用指针指示"可装入的空缓冲"和"可取出的满缓冲"。适合数据到达速率与处理速率相近、且都比较稳定的场合。
- **缓冲池（Buffer Pool）**：把缓冲区统一管理、供多个进程和设备共享（既能用于输入、又能用于输出），是现代系统采用的方式。

### 缓冲池的工作方式

缓冲池中的缓冲区按使用状态分别挂在三个队列上：空闲缓冲队列 emq、输入队列 inq、输出队列 outq。围绕它有四种工作过程：

| 工作过程 | 含义 |
|---|---|
| **收容输入** | 输入设备把数据装入从空闲队列取得的缓冲区，装满后挂到输入队列 |
| **提取输入** | 进程从输入队列取一个满缓冲区，用完后送回空闲队列 |
| **收容输出** | 进程从空闲队列取缓冲区装入输出数据，装满后挂到输出队列 |
| **提取输出** | 输出设备从输出队列取一个满缓冲区输出，用完后送回空闲队列 |

*给设备读入时间 T、缓冲区传送到用户区时间 M、CPU 处理时间 C。处理一块数据的事件单缓冲一般为 max(C, T)+M，双缓冲约为 max(C, T)*

## I/O软件独立性

I/O 软件被组织成**自下而上的若干层**，每层完成相对独立的功能，下层为上层屏蔽细节：

| 层次（从上到下） | 主要职责 |
|---|---|
| **用户层 I/O 软件** | 提供库函数/系统调用接口（如 printf、read），发起 I/O 请求；SPOOLing 也在此层之上 |
| **设备独立性软件** | 实现与设备无关的统一处理：逻辑设备名映射、设备分配、缓冲管理、错误处理、提供统一接口 |
| **设备驱动程序** | 与具体硬件打交道，把上层的抽象请求翻译成对**设备控制器寄存器**的具体操作 |
| **中断处理程序** | 处理 I/O 完成时设备发来的中断，唤醒等待的进程 |
| **硬件** | 设备控制器与设备本身 |

### 设备独立性（设备无关性）

**设备独立性**：用户编程时使用的是**逻辑设备名**而非具体的**物理设备名**，应用程序与具体物理设备**无关**。

- **好处**：① 程序可移植性强，更换设备不必改程序；② 提高设备分配的灵活性（系统可在多个同类设备中任选一个分配）；③ 便于实现 I/O 重定向。
- **实现关键**：系统维护一张**逻辑设备表（LUT，Logical Unit Table）**，记录"逻辑设备名 → 物理设备名（或驱动程序入口）"的映射。进程用逻辑名请求，系统查 LUT 转换成物理设备。

### 设备分配的数据结构

为了管理和分配设备，系统维护四张表：

| 数据结构 | 全称 | 作用 |
|---|---|---|
| **SDT** | 系统设备表 | 全系统一张，登记所有设备，指向各 DCT |
| **DCT** | 设备控制表 | 每个设备一张，记录设备状态、指向其控制器 COCT、等待队列 |
| **COCT** | 控制器控制表 | 每个控制器一张，记录控制器状态、指向其通道 CHCT |
| **CHCT** | 通道控制表 | 每个通道一张，记录通道状态及与其相连的控制器 |

分配一个设备，要依次分配/检查 **设备(DCT) → 控制器(COCT) → 通道(CHCT)** 三级，三者都可用，I/O 路径才打通。

### SPOOLing 技术（假脱机）

**SPOOLing（Simultaneous Peripheral Operation On-Line，外围设备联机同时操作）** 用软件方法把**独占设备改造成可共享的虚拟设备**，从而提高利用率。最典型应用是**多个进程共享一台打印机**。

**组成：**

- **输入井 / 输出井**：在**磁盘**上开辟的两块存储区域，分别模拟"输入设备"和"输出设备"，用来暂存数据。
- **输入缓冲区 / 输出缓冲区**：内存中的缓冲，缓和磁盘井与设备的速度差。
- **输入进程 / 输出进程**：模拟脱机 I/O 时的外围控制机，负责在井与设备间搬运数据。

**以打印为例的工作过程：** 进程要打印时，并不真的独占打印机，而是把要打印的数据**送到磁盘上的输出井**，并排入打印请求队列；真正的打印机由**输出进程**统一调度，依次从输出井取数据打印。这样每个进程"感觉"自己拥有一台打印机，实际它们共享同一台物理打印机。

**SPOOLing 的特点：**

- 提高了 I/O 速度（进程与"虚拟设备"间的数据交换是对**磁盘井**操作，快）。
- 把**独占设备改造为共享设备**（一台打印机被多个进程共享）。
- 实现了**虚拟设备**功能（每个进程都以为自己独占一台设备）。

*这一节是设备管理的"软件总览"。两条主线要拎清：① 分层 + 设备独立性 = 用统一逻辑接口屏蔽硬件差异；② SPOOLing = 用磁盘井把独占设备"虚拟"成共享设备。SPOOLing 常与"虚拟设备""为什么能共享打印机"一起考。*

## 磁盘基本知识

磁盘是最重要的**块设备**和外存，理解它的物理结构才能理解寻道算法。

### 磁盘的物理结构

- **盘面（Platter）**：磁盘由多个盘片叠成，每个盘片有上下两个**盘面**，每面对应一个**磁头**。
- **磁道（Track）**：每个盘面上一圈圈同心圆，每一圈是一条磁道，最外圈编号为 0。
- **扇区（Sector）**：每条磁道又被划分成若干段弧，每段叫一个扇区，是磁盘**读写的最小单位**（通常 512B）。
- **柱面（Cylinder）**：所有盘面上**半径相同**的磁道，纵向叠起来构成一个柱面。各盘面的磁头是联动的，处在同一柱面上。

**磁盘地址** 通常表示为 **(柱面号, 磁头号/盘面号, 扇区号)**。为了减少寻道，数据一般**先按柱面、再按磁头、最后按扇区**的顺序连续存放——填满一个柱面再换柱面，这样可以不移动磁头就读完整个柱面。

### 磁盘访问时间（重点公式）

完成一次磁盘读写的时间由三部分组成：

$$ T_{访问} = T_{寻道} + T_{旋转延迟} + T_{传输} $$

1. **寻道时间 $T_s$（Seek Time）**：磁头**移动到目标磁道**所花的时间。它取决于移动了多少磁道，是**调度算法主要优化的对象**。常用模型：$T_s = m \times n + s$（$n$ 为跨越磁道数，$m$ 为每道耗时，$s$ 为启动时间）。
2. **旋转延迟 $T_r$（Rotational Latency）**：磁头定位到磁道后，**等待目标扇区转到磁头下方**的时间。平均旋转延迟 = **转半圈的时间** = $\dfrac{1}{2} \times \dfrac{60}{r}$（$r$ 为磁盘转速，单位 r/min）。
3. **传输时间 $T_t$（Transfer Time）**：把数据**读出/写入**的时间。$T_t = \dfrac{b}{r \cdot N}$，其中 $b$ 为读写字节数，$N$ 为一条磁道上的字节数，$r$ 为转速。

*三部分里，**寻道时间最大、最值得优化**，旋转延迟其次，传输时间通常最小。所以磁盘调度（寻道算法）的目标就是**尽量减少磁头移动（寻道）**。*

**例**：磁盘转速 7200 r/min（即每转 $60/7200 \approx 8.33$ ms），则平均旋转延迟 ≈ 4.17 ms（转半圈）。

## 寻道算法

当有多个磁盘 I/O 请求在等待时，按什么顺序为它们服务，才能让磁头移动的总距离（寻道时间）最小。下面假设当前有一批待访问的磁道号请求。

**先来先服务 FCFS**

按请求到达的先后顺序依次服务，不做任何优化。

- 优点：公平，不会饿死任何请求。
- 缺点：完全不考虑磁道位置，磁头可能来回大幅横跳，平均寻道距离大。

---

**最短寻道时间优先 SSTF**

每次选择距离当前磁头位置最近的那个请求服务。

- 优点：寻道距离明显比 FCFS 短，性能较好。
- 缺点：可能产生饥饿——如果新请求不断落在磁头附近，远处的请求会被长期"插队"而饿死。不能保证平均寻道时间最短。

---

**扫描算法 SCAN**

磁头只朝一个方向移动，沿途顺路服务所有请求，到达该方向的尽头（或最远请求）后再掉头反向扫描，如同电梯上下楼。

- 兼顾了寻道效率和公平性，避免了饥饿。
- "电梯算法"得名于它像电梯一样一个方向扫到底再返回。

**循环扫描 C-SCAN**

SCAN 的改进。磁头只在一个方向扫描并服务，到达尽头后直接快速返回起点（返回途中不服务任何请求），再从头开始下一轮单向扫描。

- 目的：让各磁道的等待时间更均匀。SCAN 中刚被服务过的边缘磁道要等磁头折返很久才能再被服务，而 C-SCAN 把磁盘看成首尾相接的环，服务更公平。

---

**算法对比**

| 算法 | 选择策略 | 是否会饥饿 | 优缺点 |
|---|---|---|---|
| FCFS | 按到达顺序 | 否 | 公平但效率低 |
| SSTF | 最近磁道优先 | **可能** | 效率较高，但可能饿死远处请求 |
| SCAN | 单向扫到端点再折返 | 否 | 电梯式，兼顾效率与公平 |
| C-SCAN | 单向服务，到端点跳回起点 | 否 | 等待更均匀 |
