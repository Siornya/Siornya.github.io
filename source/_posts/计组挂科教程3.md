## 存储器层次结构概述

### 为什么需要多层存储？

理想的存储器应该：速度快、容量大、价格低——但这三点鱼与熊掌不可兼得。于是计算机用**多层次存储结构**来折中：让整体看起来"又快又大又便宜"。

```
        ┌──────────┐
        │  寄存器  │  ← 最快，最贵，容量极小（几十个）
        ├──────────┤
        │  Cache   │  ← 很快，较贵，容量小（KB~MB）
        ├──────────┤
        │   主存   │  ← 较快，适中，容量中等（GB）
        ├──────────┤
        │   辅存   │  ← 慢，便宜，容量大（TB）
        └──────────┘
     速度↑ 造价↑        容量↑ 离CPU距离↑
```

| 层次 | 典型部件 | 速度 | 容量 | 掉电后数据 |
|------|----------|------|------|-----------|
| 寄存器 | CPU 内部寄存器 | 最快（1 周期） | 几十字节 | 丢失 |
| Cache | SRAM | 几纳秒 | KB ~ MB | 丢失 |
| 主存 | DRAM（内存条） | 几十纳秒 | GB | 丢失 |
| 辅存 | 硬盘、SSD | 毫秒级 | TB | **保留** |

> **易失性（Volatile）：** 寄存器、Cache、主存断电后数据全丢，统称**易失性存储器**。辅存（硬盘等）断电不丢，是**非易失性存储器**。

## 主存储器

### 概述

主存储器（内存）是 CPU 可以直接寻址、直接读写的存储器。程序运行时，代码和数据必须先加载到主存，CPU 才能执行。

**主存的基本组成**

```
        CPU
         |
    地址总线（MAR）──→ 地址译码器 ──→ 选中某行存储单元
    数据总线（MDR）←──────────────── 读出/写入数据
    控制总线       ──→ 读写控制
```

- **MAR（存储器地址寄存器）**：存放要访问的地址，位数决定寻址范围。MAR 为 n 位，最多寻址 2ⁿ 个单元。
- **MDR（存储器数据寄存器）**：存放读出或将要写入的数据，位数 = 存储字长。

**几个基本指标：**

| 指标 | 含义 |
|------|------|
| **存储容量** | 主存能存多少数据，= 存储单元数 × 每单元位数 |
| **存储周期** | 连续两次访问存储器的最短时间间隔 |
| **存取时间** | 从发出读写命令到完成操作的时间 |
| **存储带宽** | 单位时间内传输的数据量（B/s） |

### 随机存取存储器RAM

**SRAM（静态 RAM）**

- 用触发器存储每一位，只要不断电数据就一直在
- 速度极快，但面积大、功耗高、价格贵
- 用途：Cache、CPU 内部寄存器堆

**DRAM（动态 RAM）**

- 用电容存储每一位，电容会漏电，需要定期**刷新（Refresh）**
- 速度较慢，但面积小、功耗低、价格便宜
- 用途：内存条（我们平时说的"内存"就是 DRAM）

| 对比 | SRAM | DRAM |
|------|------|------|
| 存储元件 | 触发器 | 电容 |
| 是否需要刷新 | 不需要 | **需要** |
| 速度 | 快 | 慢 |
| 集成度 | 低 | 高 |
| 价格 | 贵 | 便宜 |
| 用途 | Cache | 主存 |

### 只读存储器ROM

ROM 断电不丢数据，通常用来存放固件（如 BIOS）。

| 类型 | 特点 |
|------|------|
| **ROM** | 出厂时写入，不可更改 |
| **PROM** | 可一次性编程写入，之后不可改 |
| **EPROM** | 用紫外线照射可擦除，再用高压写入 |
| **EEPROM** | 电可擦写，可按字节操作 |
| **Flash（闪存）** | EEPROM 改进版，按块擦写，速度快，U 盘/SSD 都用这个 |

### 主存扩展

单块存储芯片容量有限，实际系统需要**位扩展**或**字扩展**来达到所需容量。

**位扩展（增加字长）**

每个地址对应的数据位数不够，把多块芯片并联，共用地址线，数据线各自负责不同的位。

```
例：用 4 片 1K×4位 的芯片 → 组成 1K×16位 的存储器
    4 片芯片数据线拼在一起，地址线共用
```

**字扩展（增加容量）**

地址空间不够，把多块芯片串联，用片选信号（CS）区分哪块芯片工作。

```
例：用 4 片 1K×8位 的芯片 → 组成 4K×8位 的存储器
    4 片芯片地址范围各占一段：0~1K、1K~2K、2K~3K、3K~4K
    用高位地址线译码后接各芯片的 CS 引脚
```

**字位同时扩展**：上面两种方式结合使用。

### 存储器与 CPU 的连接

```
地址线：CPU 高位地址 ──→ 片选译码器 ──→ 各芯片 CS
        CPU 低位地址 ──→ 直连各芯片地址引脚

数据线：CPU 数据总线 ←──→ 各芯片数据引脚（按位扩展并联）

控制线：CPU 读写信号 ──→ 各芯片 WE/OE
```

低位地址接芯片内部寻址，高位地址用来选片。

## 高速缓冲存储器

### 为什么用Cache

CPU 速度远快于主存，CPU 每次都去主存取数据，等待时间会严重拖慢速度。Cache 是夹在 CPU 和主存之间的小容量高速存储器，利用**程序局部性原理**让大多数访问都命中 Cache，从而大幅提速。

**程序局部性原理：**

- **时间局部性**：刚被访问的数据，近期很可能再次被访问（如循环变量）
- **空间局部性**：被访问的数据，其附近的数据也很可能被访问（如数组）

### Cache 的工作原理

```
CPU 发出地址
     ↓
先查 Cache ──→ 命中（Hit）  ──→ 直接从 Cache 读写，很快
     ↓
  未命中（Miss）
     ↓
从主存取出包含该地址的整个「块」
     ↓
将该块装入 Cache，同时送给 CPU
```

**命中率 h** = Cache 命中次数 / 总访问次数

**平均存取时间** = h × t_cache + (1 - h) × t_主存

> **命中率越高，平均存取时间越接近 Cache 速度。** 实际系统中命中率通常可达 90%~99%。

### Cache 与主存的地址映射

主存被划分为若干**块（Block）**，Cache 也分为若干**行（Line）**，映射方式决定主存的某块可以放到 Cache 的哪一行。

**直接映射（Direct Mapping）**

每个主存块只能放到 Cache 中固定的一行：Cache 行号 = 主存块号 mod Cache 总行数

```
主存块 0  → Cache 行 0
主存块 1  → Cache 行 1
   ...
主存块 N  → Cache 行 0   ← 与块0冲突！
主存块 N+1→ Cache 行 1
```

- 优点：硬件简单，查找速度快
- 缺点：冲突多，Cache 利用率低

**全相联映射（Fully Associative）**

主存的任何一块可以放到 Cache 的任何一行，完全自由。

- 优点：灵活，冲突少，Cache 利用率高
- 缺点：查找时要比较所有行的标记，硬件复杂，速度慢

**组相联映射（Set Associative）**（最常用）

Cache 分成若干**组**，每组有 k 行（k 路组相联）。主存块先确定映射到哪个组（直接映射），组内可以放到任意一行（全相联）。

```
组号 = 主存块号 mod Cache 组数
组内：全相联（k 路中任选一行）
```

- 兼顾了直接映射的速度和全相联的灵活性
- 实际 CPU 通常用 4 路或 8 路组相联

| 映射方式 | 灵活性 | 冲突 | 硬件复杂度 |
|----------|--------|------|-----------|
| 直接映射 | 最低 | 最多 | 最简单 |
| 全相联 | 最高 | 最少 | 最复杂 |
| 组相联 | 居中 | 居中 | 居中 |

### 替换算法

Cache 满了，新块进来时要踢走哪一行？

| 策略 | 全称 | 做法 | 特点 |
|------|------|------|------|
| **FIFO** | 先进先出 | 踢走最早进入 Cache 的块 | 简单，但可能踢走常用块 |
| **LRU** | 最近最少使用 | 踢走最久没被访问的块 | 最常用，符合局部性，效果好 |
| **LFU** | 最不经常使用 | 踢走访问次数最少的块 | 实现复杂，效果不稳定 |
| **随机替换** | — | 随机踢走一行 | 最简单，效果差 |

### Cache 写策略

CPU 写数据时，Cache 和主存如何保持一致？

**写命中时（数据在 Cache 中）：**

- **写直达（Write Through）**：同时写 Cache 和主存，主存始终最新，但写操作慢
- **写回（Write Back）**：只写 Cache，等该行被替换时才写回主存，速度快但主存可能暂时不一致

**写未命中时（数据不在 Cache 中）：**

- **写分配（Write Allocate）**：先把主存对应块装入 Cache，再写 Cache（配合写回使用）
- **非写分配（No Write Allocate）**：直接写主存，不装入 Cache（配合写直达使用）

```
常用组合：
  写回 + 写分配      （高性能 CPU，如现代 x86）
  写直达 + 非写分配  （简单场景）
```

## 总结

**1. SRAM vs DRAM**
SRAM 用触发器，不需要刷新，速度快，用于 Cache；DRAM 用电容，需要定期刷新，速度慢，用于主存。

**2. Cache 命中率计算**
```
平均存取时间 = h × t_c + (1 - h) × t_m
其中 h = 命中率，t_c = Cache 存取时间，t_m = 主存存取时间
```

**3. 三种地址映射方式**
直接映射（固定对应）< 组相联（折中，最常用）< 全相联（最灵活）

**4. LRU 替换策略**
最近最少使用，是效果最好的替换策略，也是考试默认答案。

**5. 存储器扩展**
- 位扩展：数据线并联，地址线共用，增加字长
- 字扩展：用高位地址译码选片，增加地址空间
- 低位地址接芯片内部，高位地址用于片选
