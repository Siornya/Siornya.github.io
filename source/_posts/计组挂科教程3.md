## 概述

### 存储器分类

**按存储介质分类**

存储介质指的是用什么物理东西来记录这一位是 0 还是 1。

| 分类         | 原理         |
| ------------ | ------------ |
| 半导体存储器 | 电子电路存储 |
| 磁表面存储器 | 磁化表面存储 |
| 磁芯存储器   | 小磁环磁化   |
| 光盘存储器   | 激光读写     |

其中只有半导体存储器是易失的

> **易失性（Volatile）：** 断电后数据丢失，称为**易失性**

**按存取方式分类**

- 存取时间与物理地址无关（随机访问）
  - 随机存储器在程序的执行过程中可读可写
  - 只读存储器在程序的执行过程中只读
- 存取时间与物理地址有关（串行访问）
  - 顺序存取存储器磁带
  - 直接存取存储器磁盘

**按在计算机中的作用分类**

- **主存储器（内存）**：CPU 能直接寻址、直接读写，存放正在运行的程序和数据。
- **Flash Memory**：包括SSD、U盘等，实际上属于ROM的一种。
- **辅助存储器（外存）**：如硬盘，容量大、断电不丢，但 CPU 不能直接访问，要先调入主存。
- **高速缓冲存储器（Cache）**：夹在 CPU 和主存之间，用来缓解两者速度差。

_在计算机组成原理中，因为RAM和ROM都可以被CPU直接寻址访问，所以都算作主存_

### 存储器的层次结构

理想的存储器应该：速度快、容量大、价格低——但这三点鱼与熊掌不可兼得。于是计算机用**多层次存储结构**来折中：让整体看起来"又快又大又便宜"。

```
        ┌──────────┐
        │  寄存器  │  ← 最快，最贵，容量极小（几十个）
        ├──────────┤
        │  Cache   │  ← 很快，较贵，容量小（KB~MB）
        ├──────────┤
        │   主存   │  ← 较快，适中，容量中等（GB）
        ├──────────┤
        │   辅存   │  ← 慢，便宜，容量大（TB）
        └──────────┘
     速度↑ 造价↑        容量↑ 离CPU距离↑
```

## 主存储器

### 概述

主存储器（内存）是 CPU 可以直接寻址、直接读写的存储器。程序运行时，代码和数据必须先加载到主存，CPU 才能执行。

**主存的基本组成**

```
        CPU
         |
    地址总线（MAR）──→ 地址译码器 ──→ 选中某行存储单元
    数据总线（MDR）←──────────────── 读出/写入数据
    控制总线       ──→ 读写控制
```

- **MAR（存储器地址寄存器）**：存放要访问的地址，位数决定寻址范围。MAR 为 n 位，最多寻址 2ⁿ 个单元。
- **MDR（存储器数据寄存器）**：存放读出或将要写入的数据，位数 = 存储字长。

**几个基本指标：**

| 指标         | 含义                                        |
| ------------ | ------------------------------------------- |
| **存储容量** | 主存能存多少数据，= 存储单元数 × 每单元位数 |
| **存储周期** | 连续两次访问存储器的最短时间间隔            |
| **存取时间** | 从发出读写命令到完成操作的时间              |
| **存储带宽** | 单位时间内传输的数据量（B/s）               |

### 随机存取存储器RAM

**SRAM（静态 RAM）**

- 用触发器存储每一位，只要不断电数据就一直在
- 速度极快，但面积大、功耗高、价格贵
- 用途：Cache、CPU 内部寄存器堆

**DRAM（动态 RAM）**

- 用电容存储每一位，电容会漏电，需要定期刷新
- 速度较慢，但面积小、功耗低、价格便宜
- 用途：内存条（我们平时说的"内存"就是 DRAM）

| 对比         | SRAM   | DRAM     |
| ------------ | ------ | -------- |
| 存储元件     | 触发器 | 电容     |
| 是否需要刷新 | 不需要 | **需要** |
| 速度         | 快     | 慢       |
| 集成度       | 低     | 高       |
| 价格         | 贵     | 便宜     |
| 用途         | Cache  | 主存     |

### 只读存储器ROM

ROM 断电不丢数据，通常用来存放固件（如 BIOS）。

| 类型              | 特点                                               |
| ----------------- | -------------------------------------------------- |
| **ROM**           | 出厂时写入，不可更改                               |
| **PROM**          | 可一次性编程写入，之后不可改                       |
| **EPROM**         | 用紫外线照射可擦除，再用高压写入                   |
| **EEPROM**        | 电可擦写，可按字节操作                             |
| **Flash（闪存）** | EEPROM 改进版，按块擦写，速度快，U 盘/SSD 都用这个 |

### 主存扩展

单块存储芯片容量有限，实际系统需要**位扩展**或**字扩展**来达到所需容量。

- **位扩展（增加字长）**：每个地址对应的数据位数不够，把多块芯片并联，共用地址线，数据线各自负责不同的位。
- **字扩展（增加容量）**：地址空间不够，把多块芯片串联，用片选信号（CS）区分哪块芯片工作。

**字位同时扩展**：上面两种方式结合使用。

### 存储器与 CPU 的连接

```
地址线：CPU 高位地址 ──→ 片选译码器 ──→ 各芯片 CS
        CPU 低位地址 ──→ 直连各芯片地址引脚

数据线：CPU 数据总线 ←──→ 各芯片数据引脚（按位扩展并联）

控制线：CPU 读写信号 ──→ 各芯片 WE/OE
```

低位地址接芯片内部寻址，高位地址用来选片。

### 存储器的校验

数据在存储和传输过程中可能因干扰发生**位翻转**。为了发现甚至纠正这种错误，需要在数据里附加一些**校验位**，这就是校验码。

#### 编码的最小距离

任意两组合法代码之间，二进制位数差异数的最小值称为编码的最小距离L

$L-1=D+C\ (D\geq C)$

其中D是检测错误的尾数，C是纠正错误的位数

#### 奇偶校验码

最简单的校验码，在数据后面加 1 位校验位，使整个码字中 1 的个数为奇数（奇校验）或偶数（偶校验）。

- 只能检测奇数个位错误，且无法定位、无法纠正（码距为 2）
- 优点是简单、开销小，常用于内存条、串口通信
- 奇偶校验只能检1位错，不能纠错

#### 汉明码

汉明码采用奇偶校验和分组校验，每个校验位负责监督一部分数据位。当某一位出错时，各校验位的"对/错"组合能直接指出出错的位置，从而纠正它。标准汉明码码距为 3，可以检测2位错误，纠正1位错误。

**第一步：确定校验位个数**

设数据位有 $n$ 位，校验位有 $k$ 位，码字总长为 $n + k$ 位。$k$ 个校验位能表示 $2^k$ 种状态，要能指出 $n + k$ 个位置中的任意一个出错，再加上"没有出错"这一种情况，所以必须满足：

$$2^k \geq n + k + 1$$

常见数据位对应的校验位个数：

| 数据位 n | 满足条件的最小 k | 码字总长 |
| -------- | ---------------- | -------- |
| 4        | 3                | 7        |
| 8        | 4                | 12       |
| 16       | 5                | 21       |
| 32       | 6                | 38       |

**第二步：确定校验位的位置**

把整个码字的位从左到右（或从右到左）编号为 $1, 2, 3, \dots$。**校验位放在编号为 2 的幂次的位置上**，即第 $1, 2, 4, 8, 16, \dots$ 位（记作 $P_1, P_2, P_4, P_8$），其余位置放数据位（记作 $D$）。

以 4 位数据 $D = D_3D_2D_1D_0$、3 位校验为例，7 位码字排列如下：

```
位号:  1    2    3    4    5    6    7
内容:  P1   P2   D3   P4   D2   D1   D0
```

**第三步：确定每个校验位监督哪些位**

看位号的二进制表示，某一位的位号二进制中哪几位是 1，它就被对应的校验位监督。

- $P_1$（$001$）监督所有位号**第 0 位为 1** 的位：1, 3, 5, 7
- $P_2$（$010$）监督所有位号**第 1 位为 1** 的位：2, 3, 6, 7
- $P_4$（$100$）监督所有位号**第 2 位为 1** 的位：4, 5, 6, 7

编码时，每个校验位取值使其监督范围内（含自己）的 1 的个数为偶数（采用偶校验）：

$$P_1 = D_3 \oplus D_2 \oplus D_0,\quad P_2 = D_3 \oplus D_1 \oplus D_0,\quad P_4 = D_2 \oplus D_1 \oplus D_0$$

**第四步：纠错——找出错的位**

接收方重新计算各校验位的偶校验结果，得到一组**校验子** $S_4S_2S_1$：

- 某组校验通过（1 的个数为偶）→ 对应 $S = 0$
- 某组校验失败（1 的个数为奇）→ 对应 $S = 1$

把 $S_4S_2S_1$ 当作二进制数读出，它的值**正好就是出错位的位号**：

- $S_4S_2S_1 = 000$ → 没有错误
- $S_4S_2S_1 = 101$ → 第 5 位出错，把第 5 位取反即可纠正

**完整例子**

要传输 4 位数据 $D = 1011$，即 $D_3D_2D_1D_0 = 1011$。

计算校验位（偶校验）：

$$P_1 = D_3 \oplus D_2 \oplus D_0 = 1 \oplus 0 \oplus 1 = 0$$
$$P_2 = D_3 \oplus D_1 \oplus D_0 = 1 \oplus 1 \oplus 1 = 1$$
$$P_4 = D_2 \oplus D_1 \oplus D_0 = 0 \oplus 1 \oplus 1 = 0$$

填入码字：

```
位号:  1    2    3    4    5    6    7
内容:  P1=0 P2=1 D3=1 P4=0 D2=0 D1=1 D0=1
```

发送的码字为 `0 1 1 0 0 1 1`。

假设传输中第 5 位由 0 翻转成 1，接收到 `0 1 1 0 1 1 1`。接收方重新校验：

- $S_1$：位 1,3,5,7 = 0,1,1,1 → 1 的个数为 3（奇）→ $S_1 = 1$
- $S_2$：位 2,3,6,7 = 1,1,1,1 → 1 的个数为 4（偶）→ $S_2 = 0$
- $S_4$：位 4,5,6,7 = 0,1,1,1 → 1 的个数为 3（奇）→ $S_4 = 1$

校验子 $S_4S_2S_1 = 101 = 5$，正好指出**第 5 位出错**，将其取反恢复为 0，数据无误。

## 高速缓冲存储器

### 为什么用Cache

CPU 速度远快于主存，CPU 每次都去主存取数据，等待时间会严重拖慢速度。Cache 是夹在 CPU 和主存之间的小容量高速存储器，利用**程序局部性原理**让大多数访问都命中 Cache，从而大幅提速。

**程序局部性原理：**

- **时间局部性**：刚被访问的数据，近期很可能再次被访问（如循环变量）
- **空间局部性**：被访问的数据，其附近的数据也很可能被访问（如数组）

### Cache 的工作原理

```
CPU 发出地址
     ↓
先查 Cache ──→ 命中（Hit）  ──→ 直接从 Cache 读写，很快
     ↓
  未命中（Miss）
     ↓
从主存取出包含该地址的整个「块」
     ↓
将该块装入 Cache，同时送给 CPU
```

**命中率 h** = Cache 命中次数 / 总访问次数

**平均存取时间** = h × t*cache + (1 - h) × t*主存

### Cache 与主存的地址映射

主存被划分为若干**块（Block）**，Cache 也分为若干**行（Line）**，映射方式决定主存的某块可以放到 Cache 的哪一行。

**直接映射**

每个主存块只能放到 Cache 中固定的一行：Cache 行号 = 主存块号 mod Cache 总行数

- 优点：硬件简单，查找速度快
- 缺点：冲突多，Cache 利用率低

**全相联映射**

主存的任何一块可以放到 Cache 的任何一行，完全自由。

- 优点：灵活，冲突少，Cache 利用率高
- 缺点：查找时要比较所有行的标记，硬件复杂，速度慢

**组相联映射**

Cache 分成若干**组**，每组有 k 行（k 路组相联）。主存块先确定映射到哪个组（直接映射），组内可以放到任意一行（全相联）。

```
组号 = 主存块号 mod Cache 组数
组内：全相联（k 路中任选一行）
```

- 兼顾了直接映射的速度和全相联的灵活性
- 实际 CPU 通常用 4 路或 8 路组相联

| 映射方式 | 灵活性 | 冲突 | 硬件复杂度 |
| -------- | ------ | ---- | ---------- |
| 直接映射 | 最低   | 最多 | 最简单     |
| 全相联   | 最高   | 最少 | 最复杂     |
| 组相联   | 居中   | 居中 | 居中       |

### 替换算法

Cache 满了，新块进来时要踢走哪一行？

| 策略         | 全称         | 做法                    | 特点                       |
| ------------ | ------------ | ----------------------- | -------------------------- |
| **FIFO**     | 先进先出     | 踢走最早进入 Cache 的块 | 简单，但可能踢走常用块     |
| **LRU**      | 最近最少使用 | 踢走最久没被访问的块    | 最常用，符合局部性，效果好 |
| **LFU**      | 最不经常使用 | 踢走访问次数最少的块    | 实现复杂，效果不稳定       |
| **随机替换** | —            | 随机踢走一行            | 最简单，效果差             |

### Cache 写策略

CPU 写数据时，Cache 和主存如何保持一致？

**写命中时（数据在 Cache 中）：**

- **写直达（Write Through）**：同时写 Cache 和主存，主存始终最新，但写操作慢
- **写回（Write Back）**：只写 Cache，等该行被替换时才写回主存，速度快但主存可能暂时不一致

**写未命中时（数据不在 Cache 中）：**

- **写分配（Write Allocate）**：先把主存对应块装入 Cache，再写 Cache（配合写回使用）
- **非写分配（No Write Allocate）**：直接写主存，不装入 Cache（配合写直达使用）

```
常用组合：
  写回 + 写分配      （高性能 CPU，如现代 x86）
  写直达 + 非写分配  （简单场景）
```

## 总结

**1. SRAM vs DRAM**
SRAM 用触发器，不需要刷新，速度快，用于 Cache；DRAM 用电容，需要定期刷新，速度慢，用于主存。

**2. Cache 命中率计算**

```
平均存取时间 = h × t_c + (1 - h) × t_m
其中 h = 命中率，t_c = Cache 存取时间，t_m = 主存存取时间
```

**3. 三种地址映射方式**
直接映射（固定对应）< 组相联（折中，最常用）< 全相联（最灵活）

**4. LRU 替换策略**
最近最少使用，是效果最好的替换策略，也是考试默认答案。

**5. 存储器扩展**

- 位扩展：数据线并联，地址线共用，增加字长
- 字扩展：用高位地址译码选片，增加地址空间
- 低位地址接芯片内部，高位地址用于片选

**6. 汉明码**

- 校验位个数 k 满足 $2^k \geq n + k + 1$
- 校验位放在第 1、2、4、8……（2 的幂次）位上
- 各校验位监督"位号二进制对应位为 1"的所有位，采用偶校验
- 接收方算出校验子 $S_4S_2S_1$，其值**就是出错位的位号**，为 0 则无错
- 标准汉明码码距 3，纠 1 位错；加 1 位总校验变 SEC-DED（码距 4），纠 1 位检 2 位（即 ECC 内存）

**7. 磁盘存取时间**

```
存取时间 = 寻道时间 + 旋转延迟 + 传输时间
平均旋转延迟 = 转一圈时间 ÷ 2
```
