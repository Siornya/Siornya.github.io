## 机器指令

### **指令的一般格式**

**操作码+地址码**

- **操作码**：规定指令的操作类型，如加法、跳转、读内存等
- **地址码**：给出操作数的地址信息，可以有 0~3 个地址字段

操作码位数固定时，能表示的指令数量有限。**扩展操作码**用地址码的高位来扩充操作码，让短地址指令拥有更多操作码编码空间。

指令字长不变的情况下，操作码短的指令，地址码字段多（三地址）；操作码长的指令，地址码字段少（零地址）。

| 类型 | 格式 | 含义 | 举例 |
|------|------|------|------|
| **三地址指令** | OP A1, A2, A3 | A3 ← A1 OP A2 | `ADD R1, R2, R3`（R3=R1+R2） |
| **二地址指令** | OP A1, A2 | A1 ← A1 OP A2 | `ADD R1, R2`（R1=R1+R2） |
| **一地址指令** | OP A1 | ACC ← ACC OP A1 | 另一个操作数隐含在累加器 ACC |
| **零地址指令** | OP | 操作数隐含（堆栈） | `PUSH`、`POP`、`NOP`、`HALT` |

### 指令字长

指令字长取决于操作码长度，地址码长度和地址码个数

- 指令字长固定

  指令字长=存储字长

- 指令字长可变

  按字节倍数变化

## 操作数类型和操作类型

### 操作数的数据类型

| 类型 | 说明 |
|------|------|
| **整数** | 无符号整数、有符号整数（补码） |
| **浮点数** | IEEE 754 单精度/双精度 |
| **字符** | ASCII 码、Unicode |
| **逻辑数** | 按位操作的 0/1 串 |
| **地址** | 指针，本身也是一种操作数 |

### 操作类型

CPU 指令集能做的事，基本可以归为以下几类：

| 类别 | 说明 | 典型指令 |
|------|------|----------|
| **数据传送** | 在寄存器、内存、I/O 之间搬数据 | MOV, LOAD, STORE, PUSH, POP |
| **算术逻辑** | 加减乘除、自增自减<br />按位逻辑操作 | ADD, SUB, MUL, DIV, INC, DEC<br />AND, OR, NOT, XOR |
| **移位操作** | 算术/逻辑/循环移位 | SHL, SHR, ROL, ROR |
| **程序控制** | 无条件转移，条件转移，调用/返回 | JMP, JZ, CALL, RET |
| **输入输出** | 与外设交换数据 | IN, OUT |

## 寻址方式

确定本条指令的操作数地址，下一条欲执行指令的指令地址

分为指令寻址和数据寻址

寻址方式回答的问题是：**指令里的地址码字段，到底怎么解读，才能找到真正的操作数？**

真正用到的操作数地址叫**有效地址（EA，Effective Address）**，不同寻址方式计算 EA 的方法不同。

### 立即寻址

**地址码字段里存的不是地址，而是操作数本身。**

```
指令：OP # N
# 是立即寻址的特征
N 是一个补码直接表示操作数
```

| 优点 | 缺点 |
|------|------|
| 不需要访问内存，速度最快 | 操作数大小受地址码位数限制，只能表示小常数 |

### 直接寻址

**地址码字段里存的是操作数的内存地址。**

```
指令：OP 寻址特征 A
A直接是有效地址
```

| 优点 | 缺点 |
|------|------|
| 简单直观，一次访存得到操作数 | 地址码位数有限，寻址范围受限 |

### 间接寻址

**地址码字段里存的是一个地址，那个地址里存的才是操作数的真实地址。**

```
指令：OP 寻址特征 A
         地址码 = 200

EA = A               ← 先去 200 取出一个地址
操作数 = Memory[EA]   ← 再去那个地址取操作数
```

| 优点 | 缺点 |
|------|------|
| 寻址范围大（地址本身可以很大） | 需要两次访存，速度慢 |

### 寄存器寻址

**操作数直接在寄存器中，地址码字段给出的是寄存器编号。**

```
指令：OP 寻址特征 Ri
         地址码 = Ri（寄存器编号）

EA：不需要访存，操作数在寄存器 R1 中
```

| 优点 | 缺点 |
|------|------|
| 不访存，速度很快；寄存器编号短，指令短 | 寄存器数量有限，不能存太多操作数 |

### 寄存器间接寻址

**寄存器里存的是操作数的内存地址（寄存器当指针用）。**

```
指令：ADD R0, [R1]
         地址码 = R1

EA = R1 的内容（假设 R1 = 500）
操作数 = Memory[500]
```

| 优点 | 缺点 |
|------|------|
| 寄存器中地址可以很大，寻址范围大；只需一次访存 | 比寄存器寻址多一次访存 |

*指针的本质：`*p` 就是寄存器间接寻址。*

### 基址寻址

**有效地址 = 基址寄存器（BR）的内容 + 地址码字段（偏移量）**

```
EA = (BR) + A
操作数 = Memory[EA]

其中 BR 是专用基址寄存器（或通用寄存器中指定一个），A 是指令中的偏移量
```

**用途：** 程序重定位。操作系统把程序加载到内存任意位置时，只要修改 BR 的值，程序内所有地址自动跟着偏移，无需修改每条指令。

### 变址寻址

**有效地址 = 变址寄存器（IX）的内容 + 地址码字段（基地址）**

```
EA = (IX) + A
操作数 = Memory[EA]

其中 A 是指令中的基地址（固定），IX 是变址寄存器（可变，程序自己修改）
```

**和基址寻址的区别：**

| | 基址寻址 | 变址寻址 |
|--|---------|---------|
| 指令中的 A | 偏移量（小，可变逻辑） | 基地址（固定，代表数组起始） |
| 寄存器的值 | 基地址（由 OS 设置） | 偏移量（由程序自己改变） |
| 主要用途 | 程序重定位 | 数组遍历、循环访问 |

**用途：** 遍历数组。A 是数组起始地址，IX 是下标，每次循环让 IX 加 1，就能依次访问 array[0], array[1], array[2]...

### 相对寻址

**有效地址 = 程序计数器（PC）的当前值 + 地址码字段（偏移量）**

```
EA = (PC) + A
```

**用途：** 跳转指令。偏移量 A 表示"从当前位置跳多少步"，正数向后跳，负数向前跳（实现循环）。

**优点：** 与程序在内存中的绝对位置无关，程序可以放在任何地方执行（位置无关代码，PIC），也是基址寻址的一种特殊形式。

### 堆栈寻址

**操作数隐含在堆栈中，通过栈顶指针 SP 自动访问，指令中不需要写地址。**

```
PUSH X：SP ← SP - 1，Memory[SP] ← X
POP  X：X ← Memory[SP]，SP ← SP + 1
```

### 各种寻址方式对比

```
寻址方式          有效地址 EA             访存次数    速度
─────────────────────────────────────────────────────────
立即寻址          操作数就在指令里           0次       最快
寄存器寻址        操作数在寄存器里           0次       最快
直接寻址          EA = A                  1次       快
寄存器间接寻址     EA = (R)                1次       快
基址寻址          EA = (BR) + A           1次       快
变址寻址          EA = (IX) + A           1次       快
相对寻址          EA = (PC) + A           1次       快
间接寻址          EA = Memory[A]          2次       慢
堆栈寻址          EA = (SP)               1次       快
```

## 指令格式举例

| 因素           | 说明                                                         |
| -------------- | ------------------------------------------------------------ |
| 指令系统兼容性 | 系列机通常要求向上兼容                                       |
| 操作类型       | 指令个数及操作难易程度                                       |
| 数据类型       | 哪些数据类型可参与操作                                       |
| 指令格式       | 指令字长是否固定，操作码位数，地址码位数，地址码个数，寻址方式类型 |
| 寻址方式       | 支持哪些指令寻址和操作数寻址                                 |
| 寄存器个数     | 寄存器越多，地址字段越长，但执行阶段访存越少                 |

## 总结

**1. 指令格式**
指令 = 操作码 + 地址码；地址码数量越少，单条指令能做的事越少，但指令字越短。

**2. 扩展操作码**
操作码短 → 地址多（三地址）；操作码长 → 地址少（零地址）。短操作码留给高频指令。

**3. 各寻址方式的有效地址（EA）**

```
立即寻址：操作数 = A（直接用，无 EA）
直接寻址：EA = A
间接寻址：EA = Memory[A]（两次访存）
寄存器寻址：操作数 = (R)（不访存）
寄存器间接：EA = (R)
基址寻址：EA = (BR) + A
变址寻址：EA = (IX) + A
相对寻址：EA = (PC) + A
```

**4. 基址 vs 变址**
- 基址：寄存器由 OS 设置（程序重定位用）
- 变址：寄存器由程序自己改（数组遍历用）

**5. RISC 的五个关键词**
定长指令、少量简单指令、只有 Load/Store 访存、大量寄存器、硬布线控制。
