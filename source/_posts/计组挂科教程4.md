## 概述

**为什么需要 I/O**

| 问题             | 具体表现                                                     |
| ---------------- | ------------------------------------------------------------ |
| **速度差异悬殊** | CPU 每秒执行数十亿条指令，机械硬盘寻道需要毫秒，速度差了约 10⁶ 倍 |
| **信号格式不同** | CPU 内部并行传输，串口设备逐位传输，格式完全不同             |
| **工作方式不一** | CPU 同步执行，外设异步工作，需要协调机制                     |

**输入输出系统的发展**

1. 早期

   分散连接，CPU和I/O设备串行工作，程序查询方式

2. 接口模块和DMA阶段

   CPU和I/O设备并行工作（中断方式/DMA方式）

3. 具有通道结构的阶段

4. 具有I/O处理机的阶段

## I/O设备

**I/O 系统的组成**

```
CPU ←—系统总线—→ I/O 接口 ←—I/O 总线—→ 外设
                 ├─ 控制逻辑
                 ├─ 数据缓冲
                 └─ 状态寄存器
```

I/O 接口（适配器）是 CPU 与外设之间的必经之路，负责速度匹配、格式转换、时序协调。CPU 只跟接口说话，不直接对外设发号施令。 

## I/O 接口

I/O 接口是连接 CPU 总线与外设之间的逻辑电路，是整个 I/O 系统的核心组件。

### 接口的功能和组成

| 功能         | 组成                   |
| ------------ | ---------------------- |
| 选址         | 设备选择电路           |
| 传送命令     | 命令寄存器，命令译码器 |
| 传送数据     | 数据缓冲寄存器         |
| 反应设备状态 | 设备状态标记           |

### I/O 系统的组成

```
    CPU      ┌ ─ ─ ─ ─ ─ ─ ─  I/O 接口  ─ ─ ─ ─ ─ ─ ─ ─ ┐     外部设备
             ┌──────────────┐  ┌────────┐  ┌───────────┐
数据线 <──>   │   数据缓冲    │  │        │  │ 设备选择电路│   <──> 数据线
地址线  ──>   │  寄存器 DBR   │  │  控制  │  ├───────────┤
命令线  ──>   ├──────────────┤  │  逻辑  │  │  命令寄存器 │    ──>  命令
状态线 <──    │  设备状态标记  │  │  电路  │  │ 和命令译码器 │   <──  状态
             └──────────────┘  └───────┘  └────────────┘
             └ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘
```

**核心概念：I/O 接口（适配器）是 CPU 与外设之间的必经之路，负责速度匹配、格式转换、时序协调。CPU 只跟接口说话，不直接对外设发号施令。** 

## 程序查询方式

```
CPU 发出 I/O 命令
       ↓
读取状态寄存器
       ↓
外设就绪了吗？ —— 否 ——→ 循环回去继续查询
       ↓ 是
传输数据
       ↓
完成
```

**核心问题：忙等待（Busy Waiting）**

CPU 在等待外设期间只是反复执行"查询"指令，什么有意义的工作都干不了。

```
CPU 利用率 ≈ 外设传输数据时间 / (外设传输时间 + CPU 等待时间)
极端情况：等待时间 >> 传输时间 → CPU 利用率趋近 0%
```

> **缺点：** CPU 长时间陷入无意义的轮询，无法处理其他任务。只适合外设速度较快、任务简单的嵌入式场景。

## 程序中断方式

CPU 发出 I/O 请求后，立即去做别的事。外设准备好后，主动向 CPU 发出中断信号，CPU 暂停当前任务，执行中断服务程序（ISR） 处理数据，完成后返回原来的工作。

### 中断处理的完整步骤

| 阶段       | 干了什么                                                   | 关键概念             |
| ---------- | ---------------------------------------------------------- | -------------------- |
| ① 中断请求 | 外设通过 INTR 信号线向 CPU 申请中断                        | 中断请求（IRQ）      |
| ② 中断响应 | CPU 在当前指令完成后，检测到中断，关中断（IF=0），保存断点 | 断点保护、IF 标志    |
| ③ 中断识别 | 找出是哪个设备发来的中断，确定 ISR 入口地址                | 中断向量，中断向量表 |
| ④ 执行 ISR | 保存现场（PUSH 寄存器），处理数据，恢复现场（POP），开中断 | 现场保护/恢复        |
| ⑤ 中断返回 | 执行 IRET 指令，回到断点处继续执行                         | IRET 指令            |

### CPU 视角时间线

```
CPU:  [程序A...] [程序A...] [===ISR中断处理===] [程序A续...]
外设: [准备中...] [准备中...] [↑发中断] [数据传输]
```

### 中断优先级与嵌套

- **中断优先级：** 多个设备同时请求时，CPU 按优先级决定先处理谁。通常：故障 > 时钟 > I/O > 软中断
- **多重中断（嵌套）：** 高优先级中断可以打断低优先级的 ISR，需要在 ISR 内开中断（IF=1）才能实现
- **中断屏蔽：** 每个中断源都有屏蔽位，通过设置屏蔽字可以控制哪些中断被暂时忽略

> **中断方式的局限：** 每次数据传输都要打断 CPU，且每次只能传**一个字**（CPU 参与每次数据搬运）。对于磁盘读写这样的大批量数据传输，中断次数太多，开销依然很大。

## DMA 方式

### 核心思路

引入 **DMA 控制器（DMAC）**，它可以在**不经过 CPU** 的情况下，直接在主存和外设之间批量传输数据。

### DMA 传输过程

```
CPU 初始化 DMA          CPU 继续执行         DMAC 窃取总线       传输完成
（设置地址、长度、方向）→  其他程序          → 周期，搬运数据  → DMAC 中断 CPU → CPU 做后续处理
```

### DMA 控制器的内部寄存器

| 寄存器                    | 作用                                                         |
| ------------------------- | ------------------------------------------------------------ |
| **内存地址寄存器（MAR）** | 记录当前操作的内存地址，每传一个字自动 +1                    |
| **字计数器（WC）**        | 记录剩余要传输的数据量，每传一个字自动 -1，**减到 0 时发中断** |
| **数据缓冲寄存器（DR）**  | 暂存从外设读来的数据（再写入内存）或从内存读来的数据（再发给外设） |
| **DMA 控制/状态**         | 记录传输方向（读/写）、传输状态、错误标志等                  |

### 总线竞争：周期挪用（Cycle Stealing）

DMA 和 CPU 都需要使用系统总线，两者会产生冲突。解决方案是**周期挪用**：DMAC 趁 CPU 不用总线（CPU 内部运算时）悄悄占用总线传数据，CPU 几乎感知不到延迟。

```
CPU:  [取指&执行] [取指&执行] [取指&执行] [取指&执行（继续）]
DMA:  [等待总线]  [传输] [等待总线]  [传输] [等待总线]  [传输] [完成→中断]
                  ↑DMA悄悄占用        ↑DMA悄悄占用
```

### DMA 与中断的区别

| 对比项       | 中断方式               | DMA 方式                 |
| ------------ | ---------------------- | ------------------------ |
| CPU 参与时机 | 每传一个字都要打断一次 | 仅在开始和结束时参与     |
| 数据搬运者   | CPU 亲自搬             | DMAC 自动搬              |
| 传输单位     | 字                     | 数据块                   |
| 适用场景     | 中低速设备             | 高速块设备（磁盘、网卡） |

## 总结

1. **中断响应的时机** — 当前**指令**执行完毕之后（不是整个程序），且中断未被屏蔽
2. **DMA 的 WC 寄存器** — 倒计数，每传一个字减 1，减到 0 时发出中断通知 CPU 传输完成
3. **周期挪用** — DMA 不是随时都能用总线，需要在 CPU 不用总线的周期"趁虚而入"
4. **中断嵌套的条件** — ISR 执行期间必须**主动开中断**（设置 IF=1），高优先级中断才能打断低优先级 ISR
5. **CPU 利用率排序** — 程序查询 < 程序中断 < DMA < 通道
