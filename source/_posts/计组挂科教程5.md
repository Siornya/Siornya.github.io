## 无符号数和有符号数

无符号数:无符号数就是没有正负之分的数，所有位全部用来表示数值大小。

有符号数：计算机要表示负数，必须用最高位（符号位）来区分正负：**0 表示正，1 表示负**。

所以`unsigned`变量的正是表示范围是`signed`变量的两倍+1

### 原码

符号位 + 真值的绝对值。

```
+7 的8位原码：0 000 0111
-7 的8位原码：1 000 0111
 0 有2个原码：+0 = 0000 0000，-0 = 1000 0000
```

**缺点：** 加减法要单独判断符号，电路复杂；0 有两个编码。

### 反码

正数：反码 = 原码
负数：反码 = 原码符号位不变，其余各位取反

```
-7 的原码：1 000 0111
-7 的反码：1 111 1000
```

**缺点：** 0 仍然有两个编码（0000 0000 和 1111 1111），实用性不强。

### 补码

正数：补码 = 原码
负数：补码 = 反码 + 1

```
-7 的反码：1 111 1000
-7 的补码：1 111 1000 + 1 = 1 111 1001
```

**负数补码的快速算法：** 从右往左找到第一个 1，该 1 和右边的 0 保持不变，左边的所有位（包括符号位）全部取反。

**补码到负数的朴素转换：** 最左边一位必定是1，代表$-2^n$，往后每位代表$2^i$，此处i表示从右往左的位数。

**补码的优点：**
- 0 只有唯一编码：0000 0000
- 加减法统一用加法实现，硬件电路简单
- n 位补码范围：**-2ⁿ⁻¹ ~ 2ⁿ⁻¹ - 1**（负数比正数多一个！）

**8 位补码范围：-128 ~ +127**，注意 -128 没有对应的正数。

### 三种编码对比

| 真值 | 原码 | 反码 | 补码 |
|------|------|------|------|
| +7 | 0000 0111 | 0000 0111 | 0000 0111 |
| +1 | 0000 0001 | 0000 0001 | 0000 0001 |
| +0 | 0000 0000 | 0000 0000 | 0000 0000 |
| -0 | 1000 0000 | 1111 1111 | （不存在，即 -128） |
| -1 | 1000 0001 | 1111 1110 | 1111 1111 |
| -7 | 1000 0111 | 1111 1000 | 1111 1001 |
| -128 | （无法表示） | （无法表示） | 1000 0000 |

### 移码

移码主要用于表示浮点数的**阶码**（指数部分）。

移码 = 补码的符号位取反（即在真值基础上加一个偏置量 2ⁿ⁻¹）

```
真值 +7，8 位补码：0000 0111 → 移码：1000 0111
真值 -7，8 位补码：1111 1001 → 移码：0111 1001
```

**移码的好处：** 全 0 表示最小值，全 1 表示最大值，便于比较绝对值大小（浮点数对阶时要比较阶码大小）。

## 数的定点表示和浮点表示

### 定点数

小数点位置固定不变，有两种约定：

**定点整数：** 小数点在最低位右边，表示整数

```
符号位 | 数值位
   0   | 1 0 1 1 0 1 0 1
       ↑ 小数点在这里右边（整数）
```

**定点小数：** 小数点在符号位右边，表示纯小数（绝对值 < 1）

```
符号位 | . | 数值位
  0   | . | 1 0 1 1 0 1 0 1
        ↑ 小数点固定在这里
```

**定点数的问题：** 表示范围太小，无法处理既很大又很小的数（如科学计算中的 6.02×10²³ 和 1.6×10⁻¹⁹）。于是有了浮点数。

### 浮点数

类比科学计数法：任何数都写成 **M × R^E** 的形式。

- **M（尾数，Mantissa）**：有效数字部分
- **E（阶码，Exponent）**：指数部分，决定小数点位置
- **R（基数）**：计算机中固定为 2

**浮点数在内存中的存储格式：**

```
┌────────┬─────────────────┬──────────────────────────────┐
│ 符号位 │    阶码 E        │         尾数 M               │
│  1 位  │    若干位        │          若干位               │
└────────┴─────────────────┴──────────────────────────────┘
```

### IEEE 754 标准浮点数

现代计算机统一使用 IEEE 754 标准，分为单精度（32位）和双精度（64位）。

```
单精度（float，32位）：
┌───┬──────────┬───────────────────────────┐
│ S │  E(8位)  │        M(23位)            │
└───┴──────────┴───────────────────────────┘
  1      8                  23

双精度（double，64位）：
┌───┬─────────────┬──────────────────────────────────────┐
│ S │   E(11位)   │              M(52位)                 │
└───┴─────────────┴──────────────────────────────────────┘
  1       11                      52
```

**IEEE 754 的关键细节：**

- 阶码 E 用**移码**表示，单精度偏置值为 127，双精度为 1023
- 尾数 M 采用**规格化**形式：整数位固定为 1，只存小数部分（隐含位）
- 实际值 = (-1)^S × 1.M × 2^(E-127)（单精度）

**例：** 把 -0.75 表示为 IEEE 754 单精度浮点数

```
第一步：转为二进制
  0.75 = 0.11(二进制)

第二步：规格化（移成 1.xxx × 2^E 的形式）
  0.11 = 1.1 × 2⁻¹

第三步：填各字段
  符号位 S = 1（负数）
  阶码 E = -1 + 127 = 126 = 0111 1110（移码）
  尾数 M = 1000...0（小数部分，1. 后面的 1，后面补 0）

结果：1  01111110  10000000000000000000000
```

### 规格化浮点数

为了保证尾数精度最大，规定尾数必须**规格化**：尾数的最高有效位不为 0。

- 补码规格化条件：符号位与最高数值位**不同**（正数最高位为 1，负数最高位为 0）
- 等价表述：`符号位 XOR 最高数值位 = 1`

```
规格化尾数举例（补码）：
  正数：0.1xxxxxxx  ← 最高数值位必须是 1
  负数：1.0xxxxxxx  ← 最高数值位必须是 0
```

## 定点运算

**核心思想：减法变加法**

计算机只有加法器，**所有减法都转成补码加法来做**：

```
A - B = A + (-B)
      = A 的补码 + B 的补码的负数
      = [A]补 + [-B]补
```

**求 [-B]补 的方法：** 对 [B]补 各位（包括符号位）全部取反，末位加 1。

> 这个操作也叫"对补码求补"，和原码变补码的方法完全一样。

### 补码加减法运算规则

```
[A + B]补 = [A]补 + [B]补
[A - B]补 = [A]补 + [-B]补
```

**完整步骤：**

1. 把操作数都转成补码
2. 做二进制加法（直接按位相加，逢二进一）
3. 符号位也参与运算，最高位产生的进位**丢弃**
4. 结果就是补码，需要时再转回真值

**例题：用补码计算 7 - 3（8位）**

```
7 的补码：0000 0111
3 的补码：0000 0011
-3 的补码：1111 1101

  0000 0111
+ 1111 1101
───────────
1 0000 0100   ← 最高位进位丢弃

结果补码：0000 0100 = +4  ✓
```

**例题：用补码计算 -7 + (-3）**

```
-7 的补码：1111 1001
-3 的补码：1111 1101

  1111 1001
+ 1111 1101
───────────
1 1111 0110   ← 最高位进位丢弃

结果补码：1111 0110
转回真值：负数，取反加一 → 0000 1010 = 10，所以结果是 -10  ✓
```

### 溢出判断

**什么是溢出？** 运算结果超出了 n 位补码能表示的范围（-2ⁿ⁻¹ ~ 2ⁿ⁻¹ - 1），就叫溢出。溢出时结果是错误的。

> **注意：** 只有**同号数相加**（正 + 正，或 负 + 负）才可能溢出；异号数相加不会溢出。

**判断方法一：单符号位法**

看两个操作数的符号位和结果的符号位：

| 操作数符号 | 结果符号 | 结论 |
|-----------|---------|------|
| 正 + 正 | 正 | 无溢出 |
| 正 + 正 | **负** | **正溢出（上溢）** |
| 负 + 负 | 负 | 无溢出 |
| 负 + 负 | **正** | **负溢出（下溢）** |
| 正 + 负 | 任意 | 无溢出 |

**判断方法二：双符号位（变形补码）**

用 2 位来表示符号位（正数 = 00，负数 = 11），运算后：

| 双符号位结果 | 含义 |
|-------------|------|
| **00** | 结果为正，无溢出 |
| **11** | 结果为负，无溢出 |
| **01** | **正溢出** |
| **10** | **负溢出** |

口诀：**双符号位相同 = 无溢出；双符号位不同 = 溢出。**

**例题：用双符号位检验溢出**

```
计算 64 + 64（8位，真值范围 -128~127，结果128溢出）

  64 的变形补码：00 0100 0000
  64 的变形补码：00 0100 0000

    00 0100 0000
  + 00 0100 0000
  ─────────────
    00 1000 0000  → 双符号位 00，无溢出？
```

等等，这里用 8 位（1位符号+7位数值）重新算：

```
 64 的变形补码：00 1000000
 64 的变形补码：00 1000000

   00 1000000
 + 00 1000000
 ────────────
   01 0000000   ← 双符号位 01 → 正溢出！ ✓
```

### 补码加减法的硬件：加法器

计算机用**全加器**实现加法。n 位加法器只需一套电路，正数加法、负数加法（补码）、减法（转为加法）全部通用，这正是补码的最大价值。

## 浮点四则运算

### 为什么浮点加减法更复杂？

定点数加减法直接对齐位数就能算，但浮点数形如 M × 2^E，**两个数的阶码不同时，就不能直接把尾数相加**，就像 1.2×10³ + 3.4×10¹ 不能直接把 1.2 和 3.4 相加一样。

**必须先把小数点对齐（对阶）**，再做尾数加减。

### 浮点加减法的完整步骤

**第一步：对阶**

**原则：小阶向大阶看齐**（阶码小的那个尾数右移）

为什么不是大阶向小阶看齐？因为右移丢失的是低位（精度损失小），左移丢失的是高位（精度损失大）。

```
设 X = 0.1101 × 2^01，Y = 0.1001 × 2^11（均为二进制）
阶差 = 11 - 01 = 10（十进制差 2）
X 的阶码小，X 的尾数右移 2 位，阶码变为 11：

X 对阶后：0.001101 × 2^11（尾数右移2位，高位补0）
Y 保持：  0.100100 × 2^11
```

---

**第二步：尾数加减**

对阶完成后，两个尾数的阶码相同，**按定点小数的补码加减法来算**。

```
接上例（尾数用补码）：
  X：0.001101
  Y：0.100100
     ─────────
  和：0.110001  × 2^11
```

---

**第三步：规格化（Normalization）**

加减后结果可能不满足规格化条件，需要调整。

**左规：** 结果尾数最高有效位为 0（精度损失），尾数左移 1 位，阶码减 1；重复直到满足规格化。

**右规：** 结果尾数发生溢出（双符号位变为 01 或 10），尾数右移 1 位，阶码加 1；右规只需要做一次。

```
左规示例：
  结果尾数为 00.011010 × 2^11（双符号位 00，最高位是 0，需要左规）
  左移 1 位：00.110100 × 2^10  （阶码 -1）
  双符号位 00，最高位是 1，满足规格化 ✓

右规示例：
  结果尾数为 01.101000 × 2^11（双符号位 01，正溢出，需要右规）
  右移 1 位：00.110100 × 2^100（阶码 +1）
  双符号位 00，满足规格化 ✓
```

---

**第四步：舍入（Rounding）**

右移时会丢失低位数据，需要对丢失的位进行舍入处理。

| 方法 | 做法 |
|------|------|
| **截断（截去）** | 直接丢弃多余位，简单但有误差 |
| **0 舍 1 入** | 丢失位最高位为 1 则末位加 1，类似十进制四舍五入 |
| **恒置 1** | 无论丢失什么，末位强制置 1 |

> 考试中常用**0 舍 1 入**（类似四舍五入）。

---

**第五步：阶码溢出判断**

- **阶码上溢（正溢出）**：结果的阶码超过最大值 → 浮点数溢出，运算出错
- **阶码下溢（负溢出）**：结果的阶码小于最小值 → 结果趋近于 0，按 **机器零** 处理（不报错）

### 浮点运算的整体流程图

```
      输入 X、Y
          ↓
       对阶（小阶右移）
          ↓
       尾数加减（补码）
          ↓
       结果溢出？ ── 是（右规需要）──→ 右规（尾数右移1位，阶码+1）
          ↓ 否
       最高有效位为0？── 是（左规需要）→ 左规（尾数左移，阶码-1，循环）
          ↓ 否（已规格化）
         舍入
          ↓
       阶码溢出？── 上溢 → 报错
          ↓ 下溢 → 置机器零
       输出结果
```

## 总结

**1. 三种编码的转换**
- 正数：原码 = 反码 = 补码
- 负数：反码 = 原码（符号位不变，其余取反）；补码 = 反码 + 1
- 快速求负数补码：从右往左找第一个 1，该 1 及右边不变，左边全取反

**2. 补码表示范围**
- n 位补码：-2ⁿ⁻¹ ~ 2ⁿ⁻¹ - 1
- 8 位补码：-128 ~ +127，注意负数比正数多一个（-128 无对应正数）

**3. 减法转补码加法**
```
A - B = [A]补 + [-B]补
求 [-B]补：对 [B]补 所有位（含符号位）取反，末位加 1
```

**4. 溢出判断**
- 双符号位：相同（00或11）= 无溢出；不同（01或10）= 溢出
- 只有同号数相加才可能溢出

**5. 浮点加减法五步**
```
对阶（小阶向大阶）→ 尾数加减 → 规格化（左规/右规）→ 舍入 → 阶码溢出判断
```

**6. 左规 vs 右规**
- 左规：结果精度不足（高位为0），尾数左移，阶码减，可能多次
- 右规：结果尾数溢出（双符号位01/10），尾数右移，阶码加，只需一次

**7. IEEE 754 单精度**
- 32位 = 1位符号 + 8位阶码（偏置127）+ 23位尾数（隐含最高位1）
- 实际值 = (-1)^S × 1.M × 2^(E-127)
