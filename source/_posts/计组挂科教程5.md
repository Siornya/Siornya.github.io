## 无符号数和有符号数

无符号数:无符号数就是没有正负之分的数，所有位全部用来表示数值大小。

有符号数：计算机要表示负数，必须用最高位（符号位）来区分正负：**0 表示正，1 表示负**。

所以`unsigned`变量的正是表示范围是`signed`变量的两倍+1

### 原码

符号位 + 真值的绝对值。

```
+7 的8位原码：0 000 0111
-7 的8位原码：1 000 0111
 0 有2个原码：+0 = 0000 0000，-0 = 1000 0000
```

**缺点：** 加减法要单独判断符号，电路复杂；0 有两个编码。

### 反码

正数：反码 = 原码
负数：反码 = 原码符号位不变，其余各位取反

```
-7 的原码：1 000 0111
-7 的反码：1 111 1000
```

**缺点：** 0 仍然有两个编码（0000 0000 和 1111 1111），实用性不强。

### 补码

正数：补码 = 原码
负数：补码 = 反码 + 1

```
-7 的反码：1 111 1000
-7 的补码：1 111 1000 + 1 = 1 111 1001
```

**负数补码的快速算法：** 从右往左找到第一个 1，该 1 和右边的 0 保持不变，左边的所有位（包括符号位）全部取反。

**补码到负数的朴素转换：** 最左边一位必定是1，代表$-2^n$，往后每位代表$2^i$，此处i表示从右往左的位数。

**补码的优点：**
- 0 只有唯一编码：0000 0000
- 加减法统一用加法实现，硬件电路简单
- n 位补码范围：**-2ⁿ⁻¹ ~ 2ⁿ⁻¹ - 1**（负数比正数多一个！）

**8 位补码范围：-128 ~ +127**，注意 -128 没有对应的正数。

### 三种编码对比

| 真值 | 原码 | 反码 | 补码 |
|------|------|------|------|
| +7 | 0000 0111 | 0000 0111 | 0000 0111 |
| +1 | 0000 0001 | 0000 0001 | 0000 0001 |
| +0 | 0000 0000 | 0000 0000 | 0000 0000 |
| -0 | 1000 0000 | 1111 1111 | 无独立表示 |
| -1 | 1000 0001 | 1111 1110 | 1111 1111 |
| -7 | 1000 0111 | 1111 1000 | 1111 1001 |
| -128 | （无法表示） | （无法表示） | 1000 0000 |

### 移码

移码主要用于表示浮点数的**阶码**（指数部分）。

移码 = 补码的符号位取反（即在真值基础上加一个偏置量 2ⁿ⁻¹）

```
真值 +7，8 位补码：0000 0111 → 移码：1000 0111
真值 -7，8 位补码：1111 1001 → 移码：0111 1001
```

**移码的好处：** 全 0 表示最小值，全 1 表示最大值，便于比较绝对值大小（浮点数对阶时要比较阶码大小）。

## 数的定点表示和浮点表示

### 定点数

- **定点整数：** 小数点在最低位右边，表示整数
- **定点小数：** 小数点在符号位右边，表示纯小数（绝对值 < 1）

但是定点数的表示范围很小，如要表示两个大小相差很大的数据需要很长的机器字长，所以引入了浮点数。

### 浮点数

类比科学计数法：任何数都写成 **M × R^E** 的形式。

- **M（尾数，Mantissa）**：有效数字部分
- **E（阶码，Exponent）**：指数部分，决定小数点位置
- **R（基数）**：计算机中取2、4、8、16等  

#### 浮点数的表示形式

一个浮点数在内存里就是把 **M × R^E** 拆成几个字段存起来：`Sf + E + M`

- **数符 Sf**：1 位，表示整个浮点数的正负（其实就是尾数的符号）
- **阶码 E**：决定小数点的位置（数的大小量级），本身又分**阶符**和**阶值**两部分
  - 阶码位数越多 → 表示**范围越大**
- **尾数 M**：存有效数字（数的精度），常用补码或原码
  - 尾数位数越多 → 表示**精度越高**
- **基数 R**：隐含约定好的（2、4、8…），不占存储位

#### 浮点数的表示范围

范围由**阶码**决定，精度由尾数决定。以阶码 m 位、尾数 n 位、基数为 2 为例，能表示的绝对值最大、最小的正数大致是：

```
        ←负数区→      0        ←正数区→
 ──┬───────────────┬────┬───────────────┬──→
   │               │    │               │
最小负数        最大负数  最小正数      最大正数
(上溢)                                   (上溢)
        靠近 0 的一圈叫"下溢"，当作机器零处理
```

- **正数最大值**：尾数全 1、阶码取最大正值时
- **正数最小值**：规格化尾数最小、阶码取最负值时

超出上界叫 **上溢**（overflow，溢出，要报错/中断）

落进 0 附近的空隙叫 **下溢**（underflow，太小了，一般按 0 处理）

> 注意：浮点数在数轴上**不是均匀分布**的——离 0 越近的数挨得越密，越远越稀疏。所以浮点数能表示的"个数"是有限的，大数之间会有"跳跃"。

#### 浮点数的规格化形式

同一个数能写成好多种形式，比如 `0.1×2³ = 1.0×2² = 0.01×2⁴`，太乱了。为了**唯一表示**并**保证尾数精度最大**，规定尾数必须**规格化**：尾数的最高有效位必须是有效数字（不能是 0）。

**规格化的判断条件**（设基数 R = 2）：

| 尾数表示 | 规格化条件 |
| --- | --- |
| 原码 | 最高数值位为 **1**（不看符号位） |
| 补码 | 符号位与最高数值位**不同**，即 `符号位 XOR 最高数值位 = 1` |

> 运算后如果尾数不满足规格化（比如变成 `0.001…`），就要**左移尾数、同时阶码减小**来重新规格化，这一步叫"向左规格化"；反之尾数溢出（`1x.x…`）则"向右规格化"。

**规格化浮点数默认有效数字前面有一个隐藏的`1`**

### IEEE 754 标准浮点数

```
┌───┬───────────────┬─────────────┐
│ S │  阶码(含阶符)  │     尾数     │
└───┴───────────────┴─────────────┘
```

尾数为规格化表示，非0的有效位为高位为1

|          | 符号位S | 阶码 | 尾数 | 总位数 |
| -------- | ------- | ---- | ---- | ------ |
| 短实数   | 1       | 8    | 23   | 32     |
| 长实数   | 1       | 11   | 52   | 64     |
| 临时实数 | 1       | 15   | 64   | 80     |

*短实数就是单精度浮点数`float`，长实数就是双精度浮点数`double`，临时实数又称80 位扩展精度浮点数，通常用于存储中间结果以减小误差，在某些平台上`long double`会使用这个类型*

## 定点运算

### 移位运算

#### 算术移位

移位不可以改变数的正负，所以要保持符号位不变，空位补什么取决于编码方式

| 真值 | 编码             | 添补 |
| --- | --- | --- |
| 正数 | 原码、补码、反码 | 0 |
| 负数 | 原码 | 0 |
|  | 补码 | 左移添0<br />右移添1 |
|  | 反码 | 1 |

#### 逻辑移位

算数移位针对有符号数，逻辑以为针对无符号数

逻辑以为无论左移还是右移，不管正负，一律补0

**移位时的精度问题**

- **右移**：丢的是**低位**，可能损失精度，但数值大体正确
- **左移**：丢的是**高位**，如果丢掉的是有效数字就会**出错（溢出）**

### 加法与减法运算

计算机只有加法器，所有减法都转成补码加法来做：

```
[A + B]补 = [A]补 + [B]补
[A - B]补 = [A]补 + [-B]补
```

**求 [-B]补 的方法：** 对 [B]补 各位（包括符号位）全部取反，末位加 1。

**完整步骤：**

1. 把操作数都转成补码
2. 做二进制加法（直接按位相加，逢二进一）
3. 符号位也参与运算，最高位产生的进位**丢弃**
4. 结果就是补码，需要时再转回真值

#### 溢出判断

溢出是指运算结果超出了 n 位补码能表示的范围（-2ⁿ⁻¹ ~ 2ⁿ⁻¹ - 1）。溢出时结果是错误的。

**判断方法一：单符号位法**

看两个操作数的符号位和结果的符号位：

| 操作数符号 | 结果符号 | 结论 |
|-----------|---------|------|
| 正 + 正 | 正 | 无溢出 |
| 正 + 正 | **负** | **正溢出（上溢）** |
| 负 + 负 | 负 | 无溢出 |
| 负 + 负 | **正** | **负溢出（下溢）** |
| 正 + 负 | 任意 | 无溢出 |

**判断方法二：双符号位（变形补码）**

用 2 位来表示符号位（正数 = 00，负数 = 11），运算后：

| 双符号位结果 | 含义 |
|-------------|------|
| **00** | 结果为正，无溢出 |
| **11** | 结果为负，无溢出 |
| **01** | **正溢出** |
| **10** | **负溢出** |

## 浮点加减运算

定点数加减法直接对齐位数就能算，但浮点数形如 M × 2^E，**两个数的阶码不同时，就不能直接把尾数相加**，必须先把小数点对齐（对阶），再做尾数加减。

- **第一步：对阶**：**原则：小阶向大阶看齐**（阶码小的尾数右移）因为右移丢失的是低位（精度损失小），左移丢失的是高位（精度损失大）。
- **第二步：尾数加减**：对阶完成后，两个尾数的阶码相同，**按定点小数的补码加减法来算**。
- **第三步：规格化**：加减后结果可能不满足规格化条件，需要调整。
  - **左规：** 结果尾数最高有效位为 0（精度损失），尾数左移 1 位，阶码减 1；重复直到满足规格化。
  - **右规：** 结果尾数发生溢出（双符号位变为 01 或 10），尾数右移 1 位，阶码加 1；右规只需要做一次。
- **第四步：舍入（Rounding）**：右移时会丢失低位数据，需要对丢失的位进行舍入处理。
   | 方法 | 做法 |
   |------|------|
   | **截断（截去）** | 直接丢弃多余位，简单但有误差 |
   | **0 舍 1 入** | 丢失位最高位为 1 则末位加 1，类似十进制四舍五入 |
   | **恒置 1** | 无论丢失什么，末位强制置 1 |
- **第五步：阶码溢出判断**
  - **阶码上溢（正溢出）**：结果的阶码超过最大值 → 浮点数溢出，运算出错
  - **阶码下溢（负溢出）**：结果的阶码小于最小值 → 结果趋近于 0，按 **机器零** 处理（不报错）

```
      输入 X、Y
          ↓
       对阶（小阶右移）
          ↓
       尾数加减（补码）
          ↓
       结果溢出？ ── 是（右规需要）──→ 右规（尾数右移1位，阶码+1）
          ↓ 否
       最高有效位为0？── 是（左规需要）→ 左规（尾数左移，阶码-1，循环）
          ↓ 否（已规格化）
         舍入
          ↓
       阶码溢出？── 上溢 → 报错
          ↓ 下溢 → 置机器零
       输出结果
```

## 总结

**1. 三种编码的转换**
- 正数：原码 = 反码 = 补码
- 负数：反码 = 原码（符号位不变，其余取反）；补码 = 反码 + 1
- 快速求负数补码：从右往左找第一个 1，该 1 及右边不变，左边全取反

**2. 补码表示范围**
- n 位补码：-2ⁿ⁻¹ ~ 2ⁿ⁻¹ - 1
- 8 位补码：-128 ~ +127，注意负数比正数多一个（-128 无对应正数）

**3. 减法转补码加法**
```
A - B = [A]补 + [-B]补
求 [-B]补：对 [B]补 所有位（含符号位）取反，末位加 1
```

**4. 溢出判断**
- 双符号位：相同（00或11）= 无溢出；不同（01或10）= 溢出
- 只有同号数相加才可能溢出

**5. 浮点加减法五步**
对阶（小阶向大阶）→ 尾数加减 → 规格化（左规/右规）→ 舍入 → 阶码溢出判断

**6. 左规 vs 右规**
- 左规：结果精度不足（高位为0），尾数左移，阶码减，可能多次
- 右规：结果尾数溢出（双符号位01/10），尾数右移，阶码加，只需一次

**7. IEEE 754 单精度**
- 32位 = 1位符号 + 8位阶码（偏置127）+ 23位尾数（隐含最高位1）
- 实际值 = (-1)^S × 1.M × 2^(E-127)
